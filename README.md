- - # 番茄小说更新监控与 AI 智能播报

    这是一个为 [AstrBot](https://github.com/Soulter/AstrBot) 编写的自动化独立插件。利用纯后台异步定时任务机制，实时监控**单本或多本**番茄小说网页状态，并在发现新章节时，调用 AstrBot 的大语言模型能力，向指定的群聊发送推送广播。

    ## ✨ 核心特性

    - **多小说矩阵监控**：只需在 WebUI 输入一长串小说 ID 即可同时监控无数本书。所有文本内容、时间戳独立持久化保存，绝不冲突！
    - **WebUI 原生配置**：全面适配 AstrBot。你可以在机器人的网页控制台中，直观地修改小说ID、拉取间隔（分钟）、管理员 QQ 等参数。
    - **智能人格接入 (Persona)**：独家支持在 WebUI 的下拉列表中**直接选择 AstrBot 内部已设定好的人格（如特定角色、客服等）**！监控服务在播报时会自动获取该人格的 Prompt 设定。
    - **书名及纯文本持久化**：自动精准抓取小说的真实书名、最新卷名、字数/章节纯文本内容，并在本地极速读写 `data.json` 存档。
    - **终极 UMO 探测兜底**：内置多协议穷举算法（包含 `default:GroupMessage:` 等前缀），且支持在会话中直接捕获 UMO，精准投递率 100%。
    
    ## ⚙️ 如何配置与安装
    
    1. 确保插件文件夹中存在 `_conf_schema.json`、`metadata.yaml` 和 `main.py`。
    2. 在机器人面板安装并重载插件。
    3. 打开 **WebUI -> 插件管理**，点击本插件的设置，你可以直接配置：
       - 你期望使用的人格。
       - 后台轮询的**分钟**间隔。
       - 需要监控的番茄小说 ID（多个 ID 用逗号分隔）。
    
    ## 🕹️ 可用控制指令
    
    > ⚠️ **强烈建议：想要监听哪个群，就在哪个群里发指令！** 系统会自动抓取当前会话上下文并绑定最准确的群标识。
    
    - **`/fanqie add`** - 绑定当前群聊接受推送。
    - **`/fanqie list`** - 查看当前正在接受播报的所有群聊。
    - **`/fanqie del`** - 解绑当前群聊的推送。
    - **`/fanqie force`** - 强制核对一次所有配置小说的状态。如果是新章节，除了向群内播报，还会将生成的 AI 播报文案私发给你预览。如果无更新则输出极客 Debug 面板，绝不调用大模型。
    - **`/fanqie get_umo`** - 获取当前会话的最精确底层通讯标识（UMO）。
    - **`/fanqie reset`** - 清除本地的所有缓存。清理后下一次触发必全群播报。
    - **`/fanqie help`** - 查看帮助。
